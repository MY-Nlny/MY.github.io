二、常见攻击手段与解题方法

1. 伪加密（False Encryption）

  • 原理：通过修改加密标志位误导解压软件
  • 检测方法：
    • 使用7-Zip打开测试
    • 查看hex编辑器中General Purpose Bit Flag字段
      • 伪加密：第0位为1（0x0001）
      • 真加密：第6位为1（0x0041）

2. 密码爆破

  • 适用场景：弱密码或已知字符集
  • 工具推荐：
    • John the Ripper（支持掩码爆破）
    • ARCHPR（图形化界面）
    • fcrackzip（Linux命令行工具）

  • 优化策略：

  • ```bash
    fcrackzip -v -u -D -p rockyou.txt target.zip

    -D 字典攻击 -p 指定字典 -u 过滤无效密码

4. CRC32碰撞

  • 适用场景：小文件（<6KB）无加密
  • 实现原理：通过CRC校验值逆向文件内容
  • 工具推荐：
    • python的zlib库
    • crc32killer（自动化脚本）

  • 示例代码：

  • ```python
    import zlib

    target_crc = 0x12345678 # 替换为实际CRC值
    for i in range(0xFFFFFFFF):

    if zlib.crc32(bytes([i])) == target_crc:
    print(bytes([i]))
    break

3. 文件修复

  • 常见问题：
    • 文件头损坏（修复签名50 4B 03 04）
    • 目录结构破坏（重建EOCD）
  • 工具推荐：
    • zip -FF 修复命令
    • 010 Editor手动修复

四、实战案例解析

案例1：伪加密+CRC碰撞

  1. 发现加密标志位为0x0001（伪加密）
  2. 修改标志位为0x0000后提取文件
  3. 对无法解压的小文件实施CRC碰撞

案例2：混合隐写

  1. 解压得到损坏的图片文件
  2. 使用foremost分离出隐藏ZIP
  3. 通过EXIF信息找到密码提示
